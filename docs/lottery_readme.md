# Lottery Flow — README

## Общий процесс
Это описание логики работы системы после завершения основной игры на мобильных устройствах и запуска лотереи на телевизоре.

---

## 1. Завершение игры
После того как игрок проходит игру на телефоне:
- появляется сообщение о завершении игры,
- отображается кнопка **«Перейти к ожиданию»**.

При нажатии игрок попадает на следующий экран.

---

## 2. Экран ожидания лотереи
Задачи экрана:
- показать текущие баллы игрока,
- отобразить количество доступных прокрутов,
- предоставить кнопку **«Купить прокрут»**.

### Стоимость
- 1 прокрут = **150 баллов**.

Игрок может тратить баллы на покупку дополнительных прокрутов.
После каждой покупки:
- баллы уменьшаются,
- количество прокрутов увеличивается.

---

## 3. Ожидание остальных игроков
Пока другие игроки проходят игру:
- все ожидают на этом экране,
- данные обновляются в реальном времени (через WebSocket или периодический опрос).

Когда последний игрок завершает игру:
- система автоматически переводит лобби в режим лотереи.

---

## 4. Старт лотереи
На ТВ появляется кнопка **«Начать лотерею»**.
На телефонах у всех участников отображается очередь: кто крутит сейчас.

---

## 5. Процесс кручения
Пример:
1. Сейчас очередь игрока 1.
2. На его телефоне появляется кнопка **«Крутить»**.
3. Игрок нажимает кнопку.
4. На ТВ запускается анимация вращения.
5. Выпадает число.
6. Результат сохраняется и обновляется у всех.

Игрок крутит столько раз, сколько у него есть прокрутов.
После завершения — очередь переходит к следующему игроку.

---

## 6. Таблица результатов
По мере прохождения отображается таблица:

```
Номер | Имя | Прокрутов | Выпавшие числа
```

Например:
```
1     | Alex | 3         | 12, 45, 3
2     | Max  | 2         | 8, 19
```
